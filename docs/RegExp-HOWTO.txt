This is a brief description of how to search for regular
expressions in disk images using PyFLAG.

1. Create your regular expressions file

   Each line should contain a keyword (beginning of line, no whitespace)
   describing the type or class of the thing being matched followed by
   whitespace followed by the regular expression describing the thing
   to be matched. For example
 
	Email [\w\.]+@[\w\.]+
	URL	(http:|https:|ftp:|mail:)\S+
	IPv4 \d+\.\d+\.\d+\.\d+
	IPv4x.x.x.21 \d+\.\d+\.\d+\.21
	continent    Antarctica

2. Load the file

   Load the regular expressions into the database:

      12:26 gmj:~/pyflag/utilities> ./load_regexps.sh -d -v $cwd/regexps.txt
      Dropping old regexps table
      Reading File /home/george/pyflag/utilities/regexps.txt
      Added class=Email,pattern=[\w\.]+@[\w\.]+
      Added class=URL,pattern=(http:|https:|ftp:|mail:)\S+
      Added class=IPv4,pattern=\d+\.\d+\.\d+\.\d+
      Added class=IPv4x.x.x.21,pattern=\d+\.\d+\.\d+\.21
      Added class=continent,pattern=Antarctica
      12:27 gmj:~/pyflag/utilities>

3. Select RegExp as one of the scan methods when loading data

4. Display the results

Notes:
  - Matches are case insensitive

Caveats:
  - Matches do not cross block boundaries
  - The scanner searches *all* data.   It may be very slow to load.
  - Results may be large.  Be careful what regular expressions you
    search for.
